Quick Function overview

Before we start, let’s quickly get over what a function is. I like to think of it as a transformer box of sorts. Whenever you put a certain item, from a certain pile (\( x \in X \)) inside the box you get something new out of it, and that is the image of our function. The set from which we ‘take’ items from is a domain, and function results are part of a set called: codomain, where \( x \in X \) and \( y = f(x) \). This will be very important for our graph properties.

\( f : X \rightarrow Y \); \( x \in X \) and \( y \in Y \)

We usually denote the domain of the function as \( D_f \), which is also the set of all originals (inputs).

The set of all images is called the range (or image) of the function or mapping, and it is denoted by \( Z_f \), keep in mind that set image or range is a SET of actual outputs the function produces, when codomain \( Y \) is just part of the definition, Of course, it holds that \( Z_f \subseteq Y \)
